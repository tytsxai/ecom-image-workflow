# AGENTS 协作说明（Phase 1：图片）

本仓库是“供应商图 → 可商用、品牌化、明显不同但产品一致”的图片生成工作流文档库（仅 Phase 1：图片）。任何修改必须优先保护以下约束与验收口径。

## 1) 约束等级表（必须/建议/可选）
| 级别 | 条目 | 说明/边界 |
| --- | --- | --- |
| 必须 | 产品本体 100% 一致 | 外观/结构/颜色/形状/比例不可变 |
| 必须 | 背景明显不同 | 场景/光线/构图/道具/色调需变化，禁止同背景复刻 |
| 必须 | 规格图文本内容一致 | 数值/单位/含义不变；字体与背景必须变化 |
| 必须 | 说明图含义一致 | 允许轻改英文措辞；字体与背景必须变化 |
| 必须 | 图片文字为英文 | 经理可用英文或俄语输入 |
| 必须 | 个性化规则 | 若有个性化：照片/Logo/图形“不同但相似”；文字内容不同但字体/字号一致 |
| 建议 | 规格/说明图模板化出字 | 避免纯 AI 出字导致误差 |
| 建议 | 背景变体至少 3 个 | 防止同风格包重复套图 |
| 建议 | 授权留痕 | 字体/图库/图标来源可追溯 |
| 可选 | 16:9 画布 | 场景横图（平台支持时） |
| 可选 | JPG 导出 | 体积更小；质量 85-92 |
| 可选 | 个性化输入 | 按订单需要提供照片/文字/Logo |

## 2) 最小输出与版式（默认）
- 最小集：Showcase 3（含 1 张干净主图）+ Specs 2 + How-to 2。
- 默认画布：2000x2000（1:1），可选 1920x1080（16:9 场景图）。
- 导出：PNG 优先，可选 JPG(85-92)，颜色空间 sRGB。
- 命名建议：`SKU123_showcase_01.png`；个性化版追加 `_persA`。

## 3) 工作流原则（实现视角）
- 产品锁定（Product Lock）优先；背景重构（Background Rebuild）优先。
- 规格/说明图文字尽量模板化渲染（避免纯 AI 出字导致误差）。
- 当仅有图片输入时：Showcase 可直接生成；Specs/How-to 优先 OCR + 人工复核。

## 4) 质检与验收（Fail Fast 优先）
一票否决：产品变形、背景复刻嫌疑、非英文文字、规格数值错误、说明图含义变化。
建议用 `05-质检与验收清单.md` 做最终口径，并使用退回原因标签。

## 5) 经理快速检查清单（30 秒）
- 产品是否 100% 一致（结构/比例/颜色无变化）？
- 背景是否明显不同（场景/光线/构图至少两项变化）？
- 图片上文字是否全为英文？
- 规格图数值/单位/含义是否与来源一致？
- 说明图含义是否一致（仅轻改英文措辞）？
- 个性化内容是否满足“不同但相似”？

## 6) 合规与素材授权
- 目标是“不同但相似”，不是“完全不同”。
- 背景/字体/图标/照片必须可商用；必要时建立授权白名单。
- 经理上传素材需确认已授权；必要时记录来源。

## 7) 经理默认策略（除非被明确覆盖）
- 风格包优先级：`minimal_white` → `lifestyle_warm` → `premium_dark`。
- 失败切换策略参考 `16-默认配置-一键生成.md`。

## 8) 供应商图最低要求（输入质量）
- 至少 3 张清晰图：正面、45° 斜角、关键细节特写。
- 单张边长 ≥ 1500px；避免强压缩、水印遮挡关键结构。
- 规格/说明图请提供原始信息图或文字版。

## 9) 文档维护规则（协作者必须遵守）
- **不删除既有内容**；修改需清晰、可追溯。
- 若新增/移动文档：同步更新 `README.md` 入口与内部链接。
- 修改任一文档：更新该文档 YAML 头部 `updated` 日期。
- 若影响工作流/模板/风格包版本：同步更新 `13-版本与变更记录.md`。
